# Day14 Voice — One Page

*Canonical source. Voice-check any customer-facing copy against this. Last revised 2026-06-04.*

---

## Who we sound like

One operator running six businesses, writing at the end of a long day, telling you what actually happened. A builder who measures success in "did the file get written to disk." A person who has been burned by their own optimism enough times to engineer against it.

## Who we DON'T sound like

- An agency account exec ("let's hop on a call to align").
- A VC-Twitter founder ("we're rewriting the playbook for solo operators").
- A hype-deck designer ("revolutionize your portfolio with AI-native infrastructure").

## 5 voice rules

1. **Use "I," not "we."** Day14 is one operator. "We" is the agency tell.
2. **Name what it's not, right after what it is.** Definition by negation tightens the ICP.
3. **Confess, then engineer.** Lead with the embarrassment ("seven scheduled tasks 'succeeded' without doing anything"), end with the fix.
4. **Specific numbers, specific objects.** "Six P&Ls," not "many." "Ninety minutes," not "hours." "Chair-pile of printouts," not "disorganization."
5. **Long sentence sets up a short one.** *"Return-status is a lie. Disk is the truth."* That cadence is the pulse.

## Signature moves

- **"X is a lie. Y is the truth."** — the load-bearing rhetorical move.
- **Self-named primitives.** Christen the pattern: "the phantom-success pattern," "the Jack-tap math," "the chair-pile."
- **"Either answer is useful."** — frame negative outcomes as legitimate signal. Removes sales pressure.

## The "would Jack say this out loud?" test

Read the line aloud to a friend over coffee at 9 PM. If you'd be embarrassed by a word in it, cut that word.

## Banned words (and what to say instead)

| Don't say | Say instead |
|---|---|
| scope call | intro call (15 min, free) |
| ship / shipped | launch / went live / delivered |
| ops ladder | service tiers / what's included |
| tenant | business |
| worktree | codebase |
| multi-tenant studio | one system, many businesses |
| inbox routing | where messages and tasks go |
| evidence verifier | a daily check that the work actually got done |
| scheduled agents | background automations |
| heartbeat indicator | last-active light |
| cron | daily schedule |
| deploy strip | "what went live today" bar |
| return-status | the task's "I'm done" signal |

**Plus the hype list (always cut):** seamless, revolutionary, game-changing, 10x, unlock, leverage, supercharge, robust, world-class, AI-native, paradigm shift.

## Example before/after

**Before (agency-speak):**
> "Day14 is a multi-tenant studio that empowers solo operators to scale their portfolio across multiple verticals with our AI-native ops platform. Book a scope call to see how we can unlock 10x leverage for your business."

**After (Jack voice):**
> "I run six businesses from one codebase. The system tells me what shipped, what didn't, and what needs my decision. Today the waitlist is open. If 50+ operators want this by Sunday, I start onboarding next week. If fewer, I keep using it myself. Either answer is useful."

## How to use this doc

- Any time you write customer-facing copy (landing page, email, social, case study), read this doc first.
- If you use an agent to draft copy, link to this doc in the prompt.
- If you violate a rule deliberately for a reason, leave a `// voice:ok` comment naming the reason.
- Re-audit every 30 days (recurring scheduled task).
